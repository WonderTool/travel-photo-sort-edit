# Security And Privacy

This project handles workflow guidance for personal photo libraries. Treat privacy and data safety as core security concerns.

## Supported Versions

The `main` branch is the supported version.

## Reporting A Problem

If you find a privacy or safety issue, open a GitHub issue with a minimal description. Do not attach private photos, RAW files, personal metadata, exact travel locations, access tokens, or other sensitive material.

For sensitive reports, use GitHub's private vulnerability reporting if it is enabled for the repository. If it is not enabled, open a public issue with only non-sensitive details and ask for a private follow-up path.

## Safety Expectations

Contributions should not:

- delete, overwrite, or move user originals without explicit approval;
- upload private photos to third-party services by default;
- strip pairing context between JPG and RAW/NEF files;
- hide mismatched or missing source files;
- encourage public sharing of private photo artifacts.
